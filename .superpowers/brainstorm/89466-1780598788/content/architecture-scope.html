How should we approach this?

A

Clean Slate Rewrite

Start fresh with a new project structure. Port functionality piece by piece from the existing code. Results in the cleanest architecture but highest risk and longest timeline.

B

Incremental Migration

Keep the current repo, restructure module by module. Move existing Android code into shared modules, add platform targets progressively. Lower risk, keeps git history.

C

Hybrid: New Structure, Port Logic

Create the new KMP project structure in this repo (replacing existing), but copy/adapt the working Kotlin code rather than rewriting from scratch. Best of both: clean structure + proven logic.

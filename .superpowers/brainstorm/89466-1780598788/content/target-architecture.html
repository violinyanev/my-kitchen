KMP Architecture: Target Structure

Target Architecture — Module Layout
PLATFORM APPS (thin entry points)
:androidApp
Android Activity
Manifest, Icons
:iosApp
SwiftUI Entry
Info.plist
:desktopApp
JVM main()
Window config
:webApp
Wasm/JS Entry
index.html
▼ All depend on ▼
:shared:ui — Compose Multiplatform screens, navigation, theme
commonMain targets: Android, iOS, Desktop, WasmJs
▼
:shared:domain — Use cases, repository interfaces, models
Pure Kotlin, no platform dependencies
▼
:shared:data — Ktor client, local DB (Room/SQLDelight), DataStore
commonMain + platform-specific engines
BACKEND (separate deployment)
:server — Ktor Server, Exposed ORM, PostgreSQL
JVM target, shares :shared:domain models
